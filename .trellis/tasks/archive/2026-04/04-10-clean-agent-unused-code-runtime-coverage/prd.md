# 清理 agent 模块未使用代码（基于运行覆盖率）

## Goal
基于实际运行覆盖率与现有测试/运行入口，识别并清理 `agent/` 模块中未被使用、无稳定公共契约依赖的代码，降低维护成本，同时保持现有行为不回退。

## Requirements
- 使用项目现有运行入口和测试，收集 `agent/` 模块的覆盖情况，避免仅靠静态搜索判断死代码。
- 优先清理未被运行触达、且没有显式公共导出或跨层契约依赖的实现、导出和辅助代码。
- 若发现低覆盖但仍属于公共契约、运行时装配点或延迟路径的代码，不直接删除，需要保留并说明原因。
- 变更范围保持在当前明确目标内，不顺手重构无关结构。
- 为删除动作补充或更新回归验证，确保行为保持稳定。

## Acceptance Criteria
- [ ] 能说明本次用于判断 `agent/` 未使用代码的运行覆盖率入口和判定依据。
- [ ] 已删除一批确认未使用且无契约依赖的 `agent/` 代码，并同步清理相关导出/引用。
- [ ] 受影响测试或新增回归测试通过，至少覆盖本次触达的运行路径。
- [ ] 对于未删除但被识别为低覆盖的代码，已有明确保留理由。

## Technical Notes
- 遵循 `agent/` 现有模块归属，不新增泛化层。
- 删除前先检查 `__all__`、facade、runtime contract、测试引用和跨层消费点。
- 运行覆盖率结果只作为证据之一，最终以契约边界和调用链分析作为删除决策补充。
